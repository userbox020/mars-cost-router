# Changelog

All notable changes to this project are documented here.

## 0.3.2 - 2026-07-21

- Add a root-first decision ladder with Premium risk precedence, Economy only
  when every low-risk condition holds, and Balanced as the delegated default.
- Separate cause-specific reviewed escalation from malformed-call correction,
  missing authority, model unavailability, and newly increased risk; model
  substitution remains an explicit root-reviewed decision.
- Add non-overlapping writer ownership, dependency ordering, child completion
  status, and root acceptance guidance.
- Add adaptable delegation playbooks, installation troubleshooting, and
  minimum-necessary-context privacy and provider-boundary guidance.
- Replace duplicated prose checks with versioned fixture-driven conformance and
  policy-derived structural validation.
- Preserve policy lane values, the static package surface, and published
  evidence scope.

## 0.3.1 - 2026-07-21

- Add optional requested return shapes for read-only locators, focused edit
  handoffs, change summaries, and severity-ordered reviews.
- Refresh the README and 2:20 video presentation with positive, claim-safe copy.
- Strengthen video-source synchronization and public-claim scope checks.
- Preserve the instruction-only package boundary: metadata, policy, and skill,
  with the existing policy schema.

## 0.3.0 - 2026-07-21

- Publish the independent Mars Cost Router package.
- Preserve explicit economy, balanced, and premium delegation settings.
- Add offline validation, a static demo, tests, and public project governance.
- Add closed, sanitized fixed-v1.2 and dated API rate-index summaries under
  `public-evidence/`; raw prompts and local identifiers remain private, and
  financial interpretation stays within the documented evidence scope.
- Clarify fixed-v1.2 provenance with separate frozen-suite and source-summary
  hashes.
- Publish the observed author, homepage, repository, license, and interface
  website metadata shape.
- Exclude Git internals from public-tree validation and pin CI actions to the
  resolved v4 and v5 commit SHAs with read-only checkout credentials.
- Keep the installed plugin instruction-only as metadata, policy, and skill.
- Add the deterministic 2:20 explainer source and hosted release video with local
  narration, original procedural audio, and selectable captions.
- Keep the final outro legible, isolate temporary browser profiles, resolve media
  paths consistently, and exclude ignored render artifacts from public scans.
